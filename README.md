В работе реализован и протестирован стандартный алгоритм HyperLogLog для оценки числа уникальных элементов в потоке. Для моделирования входных данных использовался генератор RandomStreamGen, который формирует поток строк длиной до 30 символов из латинских букв (a-z, A-Z), цифр (0-9) и символа '-', с возможностью разбиения потока на префиксы с шагом 5 процентов (момент времени t задаётся размером обработанного префикса). Для хеширования применялся HashFuncGen, возвращающий равномерно распределённые 32-битные значения. На каждом шаге t вычислялись две величины: точное число уникальных элементов F0(t) по префиксу потока и оценка N(t), полученная HyperLogLog. Далее для набора независимых потоков рассчитывались выборочные характеристики оценки: среднее значение E(N(t)) и стандартное отклонение sigma(t), а также границы интервала неопределённости E(N(t)) минус sigma(t) и E(N(t)) плюс sigma(t). Построены два графика: сравнение истинного F0(t) и оценки N(t), а также график среднего E(N(t)) с областью неопределённости между (E(N(t)) - sigma(t)) и (E(N(t)) + sigma(t)). Эксперименты проведены для разных значений B (числа бит индекса регистра), при этом теоретическая относительная ошибка порядка 1.04 / sqrt(2^B). Для B = 10 разброс заметно выше и оценка менее стабильна, для B = 14 оценка существенно стабильнее и ближе к истинному значению, что согласуется с теорией: при увеличении B растёт число регистров и снижается дисперсия, но увеличивается потребление памяти.
